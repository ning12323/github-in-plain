# 采集信息（写之前必做）

不用装任何软件。用 Codex、Claude Code 或 Cursor **自带的打开网页、读链接** 能力即可。

## 采集清单

写之前，先在心里或草稿里确认这些项：

```
- [ ] 项目名称 + 一句话是干什么的
- [ ] 许可证（免费吗、能商用吗）
- [ ] 星标 / 关注人数（大概即可）
- [ ] 最近有没有更新
- [ ] 有没有发新版本
- [ ] README 读了前几段
- [ ] 哪些信息没查到（要如实说明）
```

## 怎么查（按顺序）

### 1. 打开 GitHub 项目页

地址：`https://github.com/主人/项目名`

页面上直接能看到：

| 想知道什么 | 在页面哪里看 |
|-----------|-------------|
| 一句话介绍 | 标题下面 |
| 星标数量 | 右上角 ⭐ |
| 许可证 | 右侧 About 栏 |
| 主要用什么语言写的 | About 栏 |
| 最近更新 | About 栏或提交记录 |

### 2. 读 README

就是项目首页往下滚看到的大段说明。重点看：做什么、给谁用、能干什么。

如果能读网页原文，也可以试：

`https://raw.githubusercontent.com/主人/项目名/main/README.md`

`main` 不行就换 `master`。

**大白话模式**：读到 README 够写「三句话讲明白」就可以停了，不用继续翻代码。

### 3. 技术模式才需要的额外信息

只有用户明确要「深度解读 / 技术报告」时，才多看：

- 许可证全文
- 更新日志 CHANGELOG
- 目录里有没有 tests、docs 等

仍然**不要求**下载整个项目。

## 查不到怎么办

诚实写一句，例如：

> 「星标数和最后更新日期我这边没核实到，建议你自己打开项目页看一下。」

**绝对不要编造数字。**

## 禁止事项

- 不要运行 Python 或任何脚本
- 不要叫用户安装软件
- 大白话模式不要下载整个代码仓库
- 不要把采集结果写成 JSON 文件
