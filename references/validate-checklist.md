# 交付前自检（不用脚本，自己打勾）

全部勾上才能发给用户。

---

## 大白话模式

### 章节齐全吗？

- [ ] `## 三句话讲明白` — 有 3 条编号
- [ ] `## 打个比方` — 至少一段类比
- [ ] `## 它能帮你干什么` — 有条目列表
- [ ] `## 大概怎么运作的（不用懂代码）` — 无代码、无文件路径
- [ ] `## 五个你可能关心的问题` — 1～5 题都有
- [ ] `## 用不用管它` — 有表格 + **一句话总结**
- [ ] `## 和类似的东西比一比` — 有对比表

### 内容达标吗？

- [ ] 全文至少 2 处生活类比（像、好比、相当于、就像）
- [ ] 专业词都解释了，或根本没出现
- [ ] 正文没有 `/5` 分数、`[E1]`、文件路径
- [ ] 没核实的数据已说明，没有瞎编
- [ ] 不懂代码的人读完后，能转述「这项目是干嘛的」

### 五个问题关键词

- [ ] 1. 免费 · 2. 维护 · 3. 难不难 · 4. 类似 · 5. 要注意

有未勾项 → 补完再发。

## 发给用户的顺序（大白话）

不要打乱：

1. 三句话讲明白  
2. 打个比方  
3. 它能帮你干什么  
4. 五个你可能关心的问题  
5. 用不用管它  
6. 大概怎么运作的  
7. 和类似的东西比一比  
8. （可选）给懂技术的人
